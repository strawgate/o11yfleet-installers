Guide

Manage rollouts

A rollout delivers a new configuration version to connected collectors. Currently, rollouts are immediate — all connected agents receive the new config at once.

🚧 Upcoming: Progressive rollouts (gradual, by label), pause/resume, detailed progress tracking, and one-click rollback are in development.

Starting a rollout

To roll out a new configuration version:

  1. Navigate to the configuration in the O11yFleet portal
  2. Click Upload YAML to create a new version
  3. Click Start rollout or run ofleet config:rollout --config-id <id> to push the current version to all connected collectors immediately

What happens during a rollout

When you start a rollout, O11yFleet sends the current config content to all connected collectors via the OpAMP WebSocket connection. Collectors that accept remote configuration will:

  1. Receive the new configuration
  2. Validate YAML syntax
  3. Apply the configuration (may trigger a restart)
  4. Report the new effective config hash back to O11yFleet

Verify rollout

You can verify that collectors have applied the new version by checking the Agents list and collector logs. O11yFleet records the current config hash and can receive effective-config reports from collectors, but detailed per-agent rollout progress is not exposed yet.

Reverting to a previous version

If a rollout causes issues, you can revert by re-uploading the previous configuration YAML as a new version. This creates a new version that replaces the problematic one.

Note: There is currently no one-click rollback; you must manually upload the old config. One-click rollback is planned.

Current limitations

  • Rollouts are immediate and affect all connected collectors — no progressive delivery yet
  • No per-collector rollout status tracking (only success/failure of the push)
  • No ability to target specific subsets of collectors (e.g., by region or label)
  • No pause/resume capability
