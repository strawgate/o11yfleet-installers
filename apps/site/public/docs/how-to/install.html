Install O11yFleet

O11yFleet can be used in two ways: with hosted access when your workspace has been provisioned, or self-hosted on your own Cloudflare account for full control.

Option 1 — Hosted access

The hosted control plane runs at api.o11yfleet.com, but self-service registration is not enabled yet. Use hosted access if your workspace has already been provisioned, or contact us from the signup page to request access.

  1. Request access or sign in to an existing workspace at o11yfleet.com
  2. Follow the getting started guide to connect your first collector
  3. Use hosted access when you do not want to operate the control plane yourself

Pricing: Public pricing describes the current hosted access model, but access is still manually provisioned while self-service registration is unavailable.

Option 2 — Self-hosted

Self-hosting gives you full control over the O11yFleet control plane, running it in your own Cloudflare account. All collector data stays in your infrastructure.

Prerequisites

  • A Cloudflare account (free tier works for testing)
  • wrangler CLI installed
  • pnpm package manager
  • git

Installation steps

  1. Clone the repository 
    git clone https://github.com/strawgate/o11yfleet.git
cd o11yfleet
  2. Install dependencies 
    just install
  3. Create local dev secrets 
    cp apps/worker/.dev.vars.example apps/worker/.dev.vars

    The Worker reads O11YFLEET_API_BEARER_SECRET from apps/worker/.dev.vars. Local helper scripts load this file automatically and use the matching bearer secret for setup calls.

  4. Start the local stack 
    just dev-up

    This starts the local Cloudflare Worker at http://localhost:8787, the site at http://127.0.0.1:3000, applies local migrations, and seeds demo data.

  5. Sign in 
    Tenant: demo@o11yfleet.com / demo-password
Admin:  admin@o11yfleet.com / admin-password

    Override these defaults with O11YFLEET_SEED_TENANT_USER_EMAIL, O11YFLEET_SEED_TENANT_USER_PASSWORD, O11YFLEET_SEED_ADMIN_EMAIL, and O11YFLEET_SEED_ADMIN_PASSWORD in apps/worker/.dev.vars.

Deploying your self-hosted instance

  1. Configure Cloudflare credentials 
    export CLOUDFLARE_DEPLOY_API_TOKEN="your-api-token"
export CLOUDFLARE_DEPLOY_ACCOUNT_ID="your-account-id"
  2. Deploy the worker 
    cd apps/worker
pnpm wrangler deploy
  3. (Optional) Deploy the management UI 
    cd apps/site
pnpm build
pnpm wrangler pages deploy dist

Verify installation

Once deployed, verify your instance is working:

curl https://<your-worker>.<your-subdomain>.workers.dev/healthz

You should see a JSON response with status and timestamp fields.
