Guide

Connect a collector

Connect an OpenTelemetry Collector to O11yFleet by enabling the OpAMP extension. Collectors send heartbeats, report health metrics, and receive configuration assignments.

Prerequisites

  • An O11yFleet workspace with hosted access, or a local dev instance running
  • An OpenTelemetry Collector binary (contrib or core distribution) version 0.89.0 or later
  • An enrollment token from the O11yFleet portal or CLI

Supported collectors

OpAMP is implemented as a collector extension and is available in:

  • opentelemetry-collector-contrib — the full-featured distribution
  • opentelemetry-collector — the core distribution (OpAMP included since v0.89.0)
  • Custom builds — as long as the OpAMP extension is compiled in

Configuration

Via config file

Add the OpAMP extension to your config.yaml:

extensions:
  opamp:
    server:
      ws:
        endpoint: "wss://api.o11yfleet.com/v1/opamp"
    headers:
      Authorization: "Bearer YOUR_TOKEN_HERE"
    capabilities:
      reports_effective_config: true
      reports_health: true
      accepts_remote_config: true

receivers:
  otlp:
    protocols:
      grpc:
      http:

exporters:
  otlp:
    endpoint: "your-backend:4317"

service:
  telemetry:
    logs:
      level: "info"
  extensions: [opamp]
  pipelines:
    traces:
      receivers: [otlp]
      exporters: [otlp]

Via the install script

The install script writes the same OpAMP extension configuration for you:

curl --proto '=https' --tlsv1.2 -fsSL https://o11yfleet.com/install.sh | bash -s -- --token YOUR_TOKEN_HERE

Running on Kubernetes

Here's a minimal Kubernetes manifest to run a collector with OpAMP:

apiVersion: v1
kind: ConfigMap
metadata:
  name: collector-config
data:
  config.yaml: |
    extensions:
      opamp:
        server:
          ws:
            endpoint: "wss://api.o11yfleet.com/v1/opamp"
        headers:
          Authorization: "Bearer ${OPAMP_TOKEN}"
        capabilities:
          reports_effective_config: true
          reports_health: true
          accepts_remote_config: true
    receivers:
      otlp:
        protocols:
          grpc:
          http:
    exporters:
      otlp:
        endpoint: "your-backend:4317"
    service:
      extensions: [opamp]
      pipelines:
        traces:
          receivers: [otlp]
          exporters: [otlp]
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: otel-collector
spec:
  replicas: 1
  selector:
    matchLabels:
      app: otel-collector
  template:
    metadata:
      labels:
        app: otel-collector
    spec:
      containers:
      - name: otel-collector
        image: otel/opentelemetry-collector-contrib:latest
        args: ["--config=/etc/collector/config.yaml"]
        volumeMounts:
        - name: config
          mountPath: /etc/collector
        env:
        - name: OPAMP_TOKEN
          valueFrom:
            secretKeyRef:
              name: o11yfleet-secret
              key: enrollment-token
        ports:
        - containerPort: 4317
          name: otlp-grpc
        - containerPort: 55681
          name: opamp
      volumes:
      - name: config
        configMap:
          name: collector-config

Verify the connection

Once the collector starts, check the logs for OpAMP connection messages:

INFO    extensions/opamp.go:112  OpAMP extension started
INFO    extensions/opamp.go:145  connected to O11yFleet (wss://api.o11yfleet.com/v1/opamp)
INFO    extensions/opamp.go:178  enrolled successfully, received claim

Then in the O11yFleet portal, go to Agents. Your collector should appear within 5—10 seconds with a green healthy status.

Remote configuration behavior

O11yFleet does not currently expose separate monitor-only and managed mode toggles. Collectors that advertise remote-config support can receive pushed config after you start a rollout. A true monitor-only mode is planned.

Troubleshooting

Collector shows as "offline"

  • Check that the OpAMP endpoint is correct and uses wss://
  • Verify enrollment token hasn't expired
  • Ensure outbound WebSocket connections to port 443 are allowed
  • Look for "OpAMP connection error" in collector logs

Collector shows as "degraded"

  • Check whether the collector reported an unhealthy status or latest error
  • Review collector logs for OpAMP or configuration-apply failures
  • Detailed queue, memory, and dropped-span diagnostics are planned but not stored today

"enrollment token invalid" error

  • Tokens remain valid until revoked or until their configured expiry passes
  • If a token may have leaked, revoke it and generate a fresh one
  • Make sure the token was copied correctly (no extra spaces)
