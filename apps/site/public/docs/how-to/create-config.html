Guide

Create a configuration

A configuration is a named set of OpenTelemetry Collector YAML. Configurations are versioned — every edit creates an immutable new version that you can roll out safely.

Create a configuration

  1. Log in to the O11yFleet portal
  2. Navigate to Configurations in the sidebar
  3. Click Create configuration
  4. Enter a name (e.g., prod-ingress) and optional description
  5. Click Create

Upload a new configuration version

After creating a configuration, you'll be taken to the configuration detail page. Click Upload YAML to create a new version.

Paste your OpenTelemetry Collector configuration:

receivers:
  otlp:
    protocols:
      grpc:
      http:

processors:
  batch:
    timeout: 5s
    send_batch_size: 4096

exporters:
  otlp:
    endpoint: "api.honeycomb.io:4317"
    headers:
      "x-api-key": "${HONEYCOMB_API_KEY}"

service:
  telemetry:
    logs:
      level: "info"
  pipelines:
    traces:
      receivers: [otlp]
      processors: [batch]
      exporters: [otlp]

Click Upload & deploy. This stores the YAML as the current version and starts an immediate rollout to connected collectors.

🚧 Git-backed configurations coming soon

Future releases will allow you to link a configuration to a GitHub repository. O11yFleet will watch for changes and automatically create new versions when you merge PRs.

Configuration versions

Every change to a configuration creates a new version. Versions are immutable and identified by an ID and a content hash.

Version lifecycle:

  • Uploaded — Stored as the current version
  • Rolled out — Delivered to connected collectors when the rollout endpoint runs
  • Reported — Collectors can report the effective config they are running
  • Reverted — Old YAML is uploaded again and rolled out as a new version
