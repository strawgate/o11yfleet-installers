Reference

API endpoints

The O11yFleet API provides REST endpoints for managing configurations, agents, and enrollment tokens. Base URL: https://api.o11yfleet.com/api/v1

Base URL

https://api.o11yfleet.com/api/v1

Authentication

Programmatic API requests currently use the deployment-level API_SECRET in the Authorization header: 

Authorization: Bearer <API_SECRET>

Tenant scoping

Include your tenant ID in the X-Tenant-Id header:

X-Tenant-Id: <your-tenant-id>

Configurations

List configurations

GET /configurations

Response:

{ "configurations": [{ "id": "cfg_xxx", "name": "prod-ingress", "current_config_hash": "abc123...", "updated_at": "2026-04-28T12:00:00Z" }] }

Get configuration

GET /configurations/:id

Create configuration

POST /configurations
{
  "name": "prod-ingress",
  "description": "Ingress gateway config"
}

Note: YAML config is uploaded separately as a version.

Upload configuration YAML (create version)

POST /configurations/:id/versions
Content-Type: text/yaml

receivers:
  otlp:
    protocols:
      grpc:

List versions

GET /configurations/:id/versions

Returns versions plus the configuration's current_config_hash.

Get current YAML

GET /configurations/:id/yaml

Returns the raw YAML for the current version.

Agents

  • GET /configurations/:id/agents — list agents for a config
  • GET /configurations/:id/agents/:instance_uid — get agent details
  • GET /configurations/:id/stats — config fleet stats

Enrollment tokens

  • POST /configurations/:id/enrollment-token — create token
  • DELETE /configurations/:id/enrollment-tokens/:token_id — revoke token
  • GET /configurations/:id/enrollment-tokens — list tokens

Rollout

Config versions become current when uploaded. Call the rollout endpoint to push the current version immediately to all connected collectors. There is no strategy selection or progressive delivery yet.

Current behavior:

  • POST /configurations/:id/rollout pushes the current version to every connected agent for that config.
  • No batching, gradual delivery, targeting, or per-agent rollout record yet.

Planned:

  • Strategy and targeting options for progressive rollout workflows
  • Rollout progress resources and per-agent apply state
🚧 API extensions in progress: Rollout strategies, progress tracking, label-based targeting, and audit logs are under development.
