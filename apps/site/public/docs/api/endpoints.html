API endpoints

Current O11yFleet API routes are split into auth, tenant-scoped portal routes, admin routes, and the OpAMP WebSocket endpoint. This reference lists the public surface exposed by the Worker today.

Base URL

https://api.o11yfleet.com

Authentication

Tenant portal routes accept either a portal session cookie or Authorization: Bearer <API_SECRET> plus X-Tenant-Id. Admin routes require an admin session or the deployment bearer secret. 

Authorization: Bearer <API_SECRET>

System and auth routes

Method Path Use
GET /healthz Worker health check
POST /auth/login Create a portal session cookie
GET /auth/me Return the current session user
POST /auth/logout Delete the current session
POST /auth/seed Seed configured demo users; requires bearer secret

Tenant-scoped routes

These routes are scoped to one tenant. Browser requests get the tenant from the session; bearer-token requests must include: 

X-Tenant-Id: <your-tenant-id>
Method Path Use
GET /api/v1/tenant Get tenant metadata
PUT /api/v1/tenant Update tenant name
DELETE /api/v1/tenant Delete tenant if it has no configurations
GET /api/v1/team List tenant users
GET /api/v1/overview Tenant fleet summary and per-config stats
POST /api/v1/ai/guidance Portal AI guidance helper

Configuration routes

Method Path Use
GET /api/v1/configurations List configurations for the tenant
POST /api/v1/configurations Create a configuration group from JSON
GET/PUT/DELETE /api/v1/configurations/:id Read, update, or delete one configuration
POST /api/v1/configurations/:id/versions Upload YAML as a new content-addressed version
GET /api/v1/configurations/:id/versions List uploaded versions for one configuration
GET /api/v1/configurations/:id/yaml Return current YAML as text/yaml
POST /api/v1/configurations/:id/rollout Push the current version to connected collectors

Agent and token routes

Method Path Use
GET /api/v1/configurations/:id/agents List collectors connected to a configuration
GET /api/v1/configurations/:id/agents/:instance_uid Get one collector from the in-memory fleet state
GET /api/v1/configurations/:id/stats Return fleet counts for one configuration
POST /api/v1/configurations/:id/enrollment-token Create an enrollment token; token value is returned once
GET /api/v1/configurations/:id/enrollment-tokens List token metadata, not raw token values
DELETE /api/v1/configurations/:id/enrollment-tokens/:token_id Revoke an enrollment token

Admin routes

Admin routes require an admin session or API_SECRET.

Method Path Use
GET /api/admin/overview Platform-level tenant/config/token counts
POST /api/admin/ai/guidance Admin AI guidance helper
GET/POST /api/admin/tenants List or create tenants
GET/PUT/DELETE /api/admin/tenants/:id Read, update, or delete one tenant
GET /api/admin/tenants/:id/configurations List configurations for one tenant
GET /api/admin/tenants/:id/users List users for one tenant
POST /api/admin/tenants/:id/impersonate Create a tenant-scoped session for admin troubleshooting
GET /api/admin/health Check D1, R2, queue, and Durable Object health
GET /api/admin/plans List built-in plan limits

OpAMP route

Method Path Use
WebSocket /v1/opamp Collector OpAMP connection with enrollment token or assignment claim

Behavior notes

Config versions become current when uploaded. Call the rollout endpoint to push the current version immediately to all connected collectors. There is no strategy selection or progressive delivery yet.

Current behavior:

  • POST /api/v1/configurations/:id/rollout pushes the current version to every connected agent for that config.
  • No batching, gradual delivery, targeting, or per-agent rollout record yet.

Planned:

  • Strategy and targeting options for progressive rollout workflows
  • Rollout progress resources and per-agent apply state
Planned API extensions: Scoped tenant API keys, rollout strategies, progress tracking, label-based targeting, and audit logs are not implemented yet.
