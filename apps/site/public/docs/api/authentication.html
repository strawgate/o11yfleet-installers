API authentication

O11yFleet has three current authentication paths: portal session cookies, the deployment bearer secret for automation, and collector enrollment/assignment tokens for OpAMP WebSocket connections.

Portal session auth

The management UI signs users in with POST /auth/login. A successful login sets the secure fp_session cookie. Browser requests then use that cookie automatically.

Route Use
POST /auth/login Validate email/password and set session cookie
GET /auth/me Return the current signed-in user
POST /auth/logout Delete the server session and clear the cookie

Bearer API access

Programmatic API access currently uses one deployment-level bearer secret, API_SECRET. Tenant-scoped requests to /api/v1/* must also include X-Tenant-Id. Fine-grained tenant API keys and per-key read/write/admin permissions are not implemented yet.

Using bearer auth

curl -H "Authorization: Bearer <API_SECRET>" \
     -H "X-Tenant-Id: <tenant-id>" \
     https://api.o11yfleet.com/api/v1/configurations

Admin automation uses the same bearer secret against /api/admin/* routes. Do not expose this secret to collectors, browsers, or untrusted automation.

Seed route

POST /auth/seed creates configured demo users and is guarded by Authorization: Bearer <API_SECRET>. It is intended for local setup and controlled deployment bootstrap, not public signup.

Tenant isolation

Session requests are scoped to the signed-in user's tenant. Bearer-token requests to /api/v1/* are scoped by the required X-Tenant-Id header.

Permissions

The current backend distinguishes admin session access, tenant session access, and bearer-secret access. Bearer-secret callers are treated as admin-level automation; scoped API-key permissions are planned but not available today.

Collector auth

Collectors connect to /v1/opamp over WebSocket. First-time enrollment uses an fp_enroll_... token. After enrollment, the server returns a signed assignment claim and the collector can reconnect with that claim. The worker strips external x-fp-* headers before routing the socket internally.
