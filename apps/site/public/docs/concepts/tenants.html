Concept

Tenants & multi-tenancy

A tenant is an isolated organization. All resources — configurations, agents, tokens — belong to a tenant and are invisible to other tenants. O11yFleet is natively multi-tenant.

Single-tenant (self-hosted)

When you self-host O11yFleet, you typically have a single tenant — your organization. All resources belong to that tenant. The tenant is created automatically during setup.

Multi-tenant (SaaS)

In the hosted SaaS version, each customer account gets its own tenant. Tenants are completely isolated:

  • No shared data between tenants
  • Separate Durable Object namespaces
  • Per-tenant resource quotas (based on plan)
  • Session-based portal auth plus tenant-scoped API requests

Tenant scoping

All API requests must include the tenant context:

  • Web portal — session cookie includes tenant_id
  • API requests — X-Tenant-Id header (for programmatic access)
  • OpAMP connections — encoded in the signed claim
