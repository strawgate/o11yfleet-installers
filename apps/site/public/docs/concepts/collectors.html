Concept

Collectors

An OpenTelemetry Collector is an agent that receives, processes, and exports telemetry data. O11yFleet manages collectors via the OpAMP protocol, giving you visibility and control.

Collector identity

Each collector has a unique instance_uid — a UUID generated when the collector first starts. O11yFleet uses this UID to identify the collector across restarts. If the collector loses its state directory, a new UID is generated and O11yFleet treats it as a new agent.

Collector states

State Description
Connected WebSocket connection is active; messages received within the 90s stale threshold
Degraded Connected but the collector reported unhealthy status or an error
Disconnected WebSocket closed or no messages for >90s

Health metrics

Connected collectors report basic health information:

  • Heartbeat — seconds since last message (stale >90s -> disconnected)
  • Version — collector binary version (e.g., 0.105.0)
  • Status — connected, disconnected, or unknown
  • Healthy — boolean indicating overall health
🚧 More detailed metrics coming soon: Export queue sizes, dropped spans, memory usage, and exporter errors are planned but not yet implemented.

Labels and targeting

Coming soon: Collectors will be able to report labels (key-value pairs) for grouping and targeting. This will enable targeted rollouts to specific subsets (e.g., region=us-east).

Capabilities

Collectors advertise capabilities during enrollment:

  • Managed config — accepts pushed configurations from O11yFleet
  • Remote actions — restart, diagnostics, and binary updates are planned
  • Watch extensions — file and Git watching is planned
