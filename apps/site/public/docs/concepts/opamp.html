Concept

OpAMP protocol

OpAMP (Open Agent Management Protocol) is a W3C-community-group specification for managing observability agents at scale. O11yFleet implements the OpAMP protocol to handle collector enrollment, configuration delivery, and health monitoring.

What is OpAMP?

OpAMP is a protocol that allows a central server to manage OpenTelemetry Collectors (and other agents) remotely. It defines:

  • Enrollment — how agents authenticate and obtain long-term credentials
  • Connection — WebSocket-based bidirectional communication
  • Configuration — pushing new configs and receiving effective state
  • Health — heartbeats, capabilities, and status reporting
  • Remote actions — restart, diagnostics, and binary updates are planned

How OpAMP works

OpAMP uses a persistent WebSocket connection between the collector and the server:

Client (collector)
  1. Connect via WebSocket with enrollment token
  2. Receive a signed assignment claim
  3. Report effective config
  4. Wait for pushed config assignments
  5. Receive config content when changed
  6. Apply and restart
  7. Report new effective config
Server (O11yFleet)
  1. Authenticate enrollment token
  2. Issue signed claim
  3. Accept WebSocket connection
  4. Store connection in DO
  5. Push config when the rollout endpoint is called
  6. Track health, version, errors
  7. Verify signed claims on reconnect

OpAMP vs. OpenTelemetry

OpenTelemetry standardizes telemetry data (traces, metrics, logs). OpAMP standardizes agent management — how you configure, update, and monitor the agents that produce that data. They're complementary:

  • OpenTelemetry — what data to send, where to send it
  • OpAMP — how to manage the software that sends it

Collector implementations

OpAMP is supported in the official OpenTelemetry Collector distributions:

  • opentelemetry-collector-contrib — the full-featured distribution (recommended)
  • opentelemetry-collector — the minimal core distribution
  • Both include the opamp extension since v0.89.0

OpAMP features in O11yFleet

  • Full protocol compliance — implements OpAMP v0.4.0 specification
  • Managed config — collectors that advertise remote-config capability can receive pushed YAML after a rollout is started
  • Mode separation — monitor-only and managed modes are not implemented as separate toggles yet
  • Signed claims — HMAC-SHA256 for stateless auth verification
  • WebSocket hibernation — idle connections sleep with zero cost
  • Connection multiplexing — multiple agents per connection (future)
