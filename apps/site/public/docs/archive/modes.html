Monitor-only vs Managed mode

Archived concept page. Separate monitor-only and managed mode toggles are not implemented in the current MVP; collectors that advertise remote-config support can receive pushed config after a rollout is started.

Monitor-only mode

In monitor-only mode, O11yFleet collects health metrics and effective configuration from collectors but never pushes configuration changes. This is the safest default — you get immediate visibility without altering collector behavior.

monitor-only · collector health
What O11yFleet does
  • Receives heartbeats and health metrics
  • Stores effective configuration
  • Tracks version drift
  • Shows connection status and basic health
  • Flags connectivity issues
What O11yFleet does NOT do
  • Push configuration changes
  • Restart collectors
  • Modify collector behavior
  • Require changes to collector config

Managed mode

Managed mode enables O11yFleet to push configurations to collectors. Collectors in managed mode accept assignments, download new configs from R2, and restart automatically when configs change.

Managed mode capabilities:

  • Push configuration updates (UI or Git)
  • Immediate global rollouts today; progressive rollouts are planned
  • Manual rollback by re-uploading a previous YAML version
  • Remote restart (future)
  • Binary updates (future)

Enabling managed mode

Managed mode selection is not available in the current MVP. The planned workflow is:

  1. Go to Workspace → Settings → Modes
  2. Toggle Managed mode to enabled
  3. Set a label selector (e.g., env=staging) to limit which collectors are managed
  4. Save

Collectors matching the selector will now accept configuration pushes. Others remain in monitor-only mode.

Mixed-mode operation

It's common to run some collectors in monitor-only and others in managed mode:

  • Production gateways — managed (you control configs)
  • Developer laptops — monitor-only (developers keep control)
  • New fleets — start monitor-only, transition to managed once trusted
  • Legacy collectors — monitor-only until upgraded

No lock-in

Even in managed mode, you can always edit the collector's local config file manually. O11yFleet will detect drift and re-push the intended version on the next heartbeat, but you retain full control.

And if you ever stop using O11yFleet, just remove the opamp extension from your collector config — no other changes needed.
